# Backend Rules Overview

Focus on interface boundaries, configuration management, authentication/authorization, error handling, and maintainability.

## Architecture Principles

- Controllers are thin, service layer is clear
- DTO / schema precedes business implementation
- Environment variables and secrets are not hardcoded
- Logging, exception, and validation paths are explicit
