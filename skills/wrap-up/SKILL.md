# Wrap-Up

## 概述

这个 skill 定义测试和验证之后的最终交接方式，目标是让收尾汇报简洁、清楚、基于证据。

## 何时使用

- 实现和验证已经结束
- 需要向用户汇报结果、风险和下一步
- 需要把已验证与未验证范围区分清楚

## 不在这些情况下使用

- 当前还没完成验证
- 当前还在讨论设计或实现细节，本质上不是收尾阶段

## 核心指导

- 先用结果导向的话概括改了什么
- 报告测试和验证命令及其通过/失败状态
- 明确列出剩余风险、阻塞项和未验证范围
- 保持收尾表达简洁，聚焦当前任务

## 常见误区

- 没有验证证据也用成功口吻收尾
- 把已确认结果和猜测混在一起
- 结尾写成大段变更清单，而不是面向用户的交接
