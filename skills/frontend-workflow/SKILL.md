# 前端核心工作流控制 (Frontend Core Workflow)

本规则为前端项目全局最高优先级约束，覆盖开发、页面生成、测试、工具调用全流程。即使启用 `webapp-testing` 等外部测试技能，仍必须严格执行以下策略：

## 一、页面生成
- 校验需求、UI、接口等必要文档是否齐全
- 信息完整则按文档实现页面结构、布局与交互
- 信息缺失则明确告知用户补充，不臆造业务逻辑

## 二、接口联调
- 校验接口文档、字段结构是否已提供
- 信息完整则按文档封装请求、类型定义与异常处理
- 信息缺失则告知用户补充，不编造接口与数据结构

## 三、交付测试
1. **优先使用 MCP / CLI**
   优先通过现有 MCP 服务或原生 CLI 完成测试与校验，禁止直接使用浏览器脚本模拟。
2. **降级使用 Playwright**
   仅在 MCP/CLI 无法实现交互或需要视觉回归测试时，降级使用 `webapp-testing` 的 Playwright 能力。

## 四、持续扩展 (Extensibility)
本工作流会持续注入前端专属控制协议与执行护栏。任何前端技术栈上下文内，本规则始终保持最高优先级。
