# 前端核心工作流控制 (Frontend Core Workflow)

作为前端项目的总控工作流与守护进程，当遇到端侧测试执行任务时（即使触发并挂载了诸如 `webapp-testing` 等外部测试技能包），你必须无条件遵循以下优先级退避与调度策略：

## 工具使用优先级战略 (Tool Priority Constraints)

1. **最高级优先 (Primary)**：
   分析任务时，首先扫描并尝试利用当前集成环境中已经启动的 MCP（Model Context Protocol）服务器相关能力，或系统可用的原生 CLI 命令行工具。如果能够通过发送 MCP 指令或 CLI 得到解决，**严禁**直接调用原生浏览器脚本模拟操作。
   
2. **后备降级降级 (Fallback)**：
   仅仅在极其特殊的情境下，例如：
   - MCP 或 CLI 工具确实无法覆盖到目标页面特定的 DOM 元素交互。
   - 必须执行强视觉回归或跨域跨层级的 UI 测试流。
   此时，你才能且只被允许退回并执行通过 `webapp-testing` 挂载的底层 Playwright 测试脚本作为兜底方案。

## 持续完善 (Extensibility)
此技能随时将被注入新的前端专属控制协议与护栏限制。一旦当前上下文涉及任何前端技术栈，遵守本文件的协议永远处于最高层级优先级。
